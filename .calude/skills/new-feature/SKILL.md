# New Feature

Gate sequence. Do not skip steps.

1. Invoke grill-me. Complete the full interview. Get explicit confirmation.
2. If feature touches architecture, routing, taxonomy, or content ownership → invoke system-design skill before writing any code.
3. If feature touches UI → apply animation rules: animate state changes, transitions, loading states. Never animate destructive confirmations, form submissions awaiting response, or repeated micro-interactions after first use.
4. Write a 3-bullet implementation plan. Show it. Wait for approval.
5. Implement. One logical unit at a time. Show diffs, not full files.
6. After implementation → run wrap-up.

Abort and return to grill-me if requirements become unclear mid-implementation.
